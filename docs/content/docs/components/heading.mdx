## Why AloeUI Heading?

While standard UI libraries often couple semantic tags with visual styles, AloeUI decouples them, allowing you to maintain perfect SEO hierarchy without compromising your design system's integrity.

## Key Features

- **True Polymorphism**: Built with advanced TypeScript generics, it adapts its underlying HTML element and strictly validates attributes based on the `as` prop. Whether it's an h1 or a div, your refs and types remain 100% accurate.
- **Atomic Typography Control**: Fine-tune tracking (letter-spacing) and leading (line-height) through dedicated props. No more "fighting" with global styles for specific hero sections.
- **Intelligent Defaults**: Optimized for modern SaaS aesthetics with `tracking-tight` and `text-balance` enabled by default, ensuring your headlines look premium out of the box.
- **Layout Safety**: Includes a `truncate` prop for overflow management and `balance` for preventing "orphaned" words in multi-line headings.

## Under the Hood

For the technical audience, here is why this implementation is superior:

- **Semantic Mapping**: The component intelligently infers the HTML tag from the `size` prop if `as` is not provided. This enforces accessibility best practices by default.
- **Radix Slot Integration**: Full support for the `asChild` pattern, allowing seamless integration with animation libraries like Framer Motion or complex functional wrappers.
- **Type-Safe Ref Forwarding**: Unlike basic implementations that use `any` or loose `HTMLElement` types, this component uses a `PolymorphicRef` pattern to ensure the ref object matches the actual rendered DOM element.
